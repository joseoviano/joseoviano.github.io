Doble Arranque Windows 10 2 discos

Arranque dual en un mismo equipo con 2 discos y 2 Windows 10

Instala Windows 10 en dos discos físicos distintos y elige sistema al iniciar mediante Windows Boot Manager o el menú de arranque UEFI/BIOS. Guía paso a paso con BCD, GPT/MBR y solución de errores.

Checklist rápido

Escenarios y recomendaciones

EscenarioVentajasCómo elegir el sistema
A) Boot Manager único (recomendado) — Instalas el 2º Windows con ambos discos conectados. Menú automático al arrancar, fácil de renombrar/ordenar, restauración simple del BCD. Windows Boot Manager con tiempo de espera (timeout).
B) Sistemas aislados — Instalas cada Windows con el otro disco desconectado. Independencia total (si un disco falla, el otro arranca). No comparten ESP/BCD. Tecla de Boot Menu de la placa (F12, Esc, etc.).

En UEFI moderno, usa GPT en ambos discos. El gestor de arranque (ESP) suele residir en el disco 0; el segundo Windows añadirá su entrada al BCD existente.

Preparación

Comprueba firmware y estilo de partición
  • Win+R → msinfo32 → Modo de BIOS: UEFI o Heredado.
  • En Administración de discos: ambos discos preferible en GPT si usas UEFI.
  • Si hay BitLocker, suspéndelo antes de tocar BCD/particiones.
Licencias y nombres
  • Legalmente necesitas 1 licencia por instalación.
  • Decide nombres: p.ej., “Windows 10 Trabajo” y “Windows 10 Juegos”.
  • Ten a mano el USB de instalación creado con Media Creation Tool o Rufus.

Instalación del segundo Windows (con Boot Manager único)

  1. Conecta ambos discos y el USB de instalación. Entra al Boot Menu (F12/F11/Esc) y arranca desde el USB (UEFI).
  2. En el instalador: idioma → Instalar ahora → omite o introduce clave → elige Windows 10 Pro.
  3. Selecciona Personalizada. En la lista de discos, identifica el disco 2 por tamaño/modelo.
  4. Crea/elige una partición en el disco 2 (recomendado vacío). No borres particiones del disco 1.
  5. Continúa. El instalador copiará archivos y, al detectar otro Windows, añadirá una entrada al Windows Boot Manager.
  6. Tras el primer reinicio, no arranques de nuevo desde el USB. Completa OOBE (cuentas, red, privacidad).

Si el disco 2 no aparece: revisa modo AHCI en BIOS/UEFI y puertos NVMe/RAID. Actualiza BIOS si es muy antigua.

Instalación con sistemas aislados (opcional)

Instalar aislado y elegir por tecla de arranque
  1. Apaga el equipo y desconecta físicamente el disco 1.
  2. Instala Windows 10 en el disco 2 con el USB (creará su propia ESP/BCD).
  3. Apaga, reconecta el disco 1. Ahora cada disco arranca solo; elige al encender con F12/Esc.

Ideal para aislar entornos o pruebas. Inconveniente: no hay menú unificado.

Configurar el menú de arranque (BCD)

Desde interfaz gráfica
  1. Configuración > Sistema > Acerca de > Configuración avanzada del sistema → Inicio y recuperación → Configuración.
  2. Elige el sistema predeterminado y el tiempo de espera (p.ej., 5–10 s).
  3. En msconfig → pestaña Arranque: renombra entradas y ajusta tiempo.
Consola (avanzado) 
REM Ver entradas
bcdedit /enum

REM Mostrar menú y tiempo
bcdedit /set {bootmgr} displaybootmenu yes
bcdedit /timeout 8

REM Renombrar la entrada actual
bcdedit /set {current} description "Windows 10 Trabajo"

REM Si falta la entrada del 2º Windows, puedes añadirla:
REM Sustituye {GUID} por el que devuelve /copy y E: por la partición del 2º Windows
bcdedit /copy {current} /d "Windows 10 Juegos"  
REM Copia devuelve un GUID, úsalo en:
bcdedit /set {GUID} device partition=E:
bcdedit /set {GUID} osdevice partition=E:
bcdedit /set {GUID} path \Windows\System32\winload.efi
bcdedit /set {GUID} systemroot \Windows

En BIOS heredada usa winload.exe en lugar de winload.efi.

UEFI/BIOS: orden de arranque y teclas útiles

  • Entra en BIOS/UEFI (Del/F2) y pon Windows Boot Manager como primera opción.
  • Menú de arranque rápido: F12 (Dell/Lenovo/Acer), Esc→F9 (HP), F11 (MSI), F8/F12 (ASUS, según modelo).
  • Desactiva temporalmente Fast Boot si no ves dispositivos.

Mantenimiento y buenas prácticas

Separación y datos
  • Guarda datos en una 3ª partición o disco para acceder desde ambos sistemas.
  • Evita hibernación cruzada (no montar NTFS de un Windows que está hibernado).
Imágenes y recuperación
  • Crea imagen del sistema de cada instalación tras dejarla a punto.
  • Guarda un USB de recuperación. Anota las letras de unidad de cada Windows.

Solución de problemas

Arranca directo a un Windows y no aparece el menú
  • Entra a msconfig → pestaña Arranque y verifica entradas.
  • Fuerza el menú: bcdedit /set {bootmgr} displaybootmenu yes y bcdedit /timeout 8.
El segundo Windows no figura en el BCD
  • Desde el Windows que sí arranca, identifica la letra del otro (p.ej., E:).
  • Reconstruye con bcdboot: 
    bcdboot E:\Windows /addlast /d /f UEFI
    En BIOS heredada: /f BIOS o /f ALL.
BCD/MBR dañado, no arranca ninguno
  • Arranca desde el USB → Reparar el equipo → Solucionar problemas → CMD.
  • Comandos útiles: 
    bootrec /fixmbr
bootrec /fixboot  (si da acceso denegado, usa: bootsect /nt60 sys)
bootrec /scanos
bootrec /rebuildbcd
    Si sigue fallando, vuelve a inyectar cargador: 
    bcdboot C:\Windows /l es-es /s S: /f UEFI
    (C: es el Windows objetivo; S: es la partición EFI)
Conflictos con BitLocker o RAID/Intel RST
  • Suspende BitLocker antes de tocar BCD y reanúdalo después.
  • Si usas RAID/RST, instala los controladores durante la instalación o cambia a AHCI (requiere preparación previa).
Fechas y horas desincronizadas entre sistemas
  • Usa la misma zona horaria y sincronización NTP. Evita cambiar manualmente la hora en uno solo.

Atajos y utilidades

  • Win+X → Administración de discos.
  • Shift+Reiniciar → opciones avanzadas de arranque.
  • diskpart: list disk, list vol, sel disk X, convert gpt/mbr.
